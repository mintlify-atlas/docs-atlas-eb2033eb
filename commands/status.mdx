---
title: 'status'
description: 'Show vault and sync status'
---

Display comprehensive status information about your BlankDrive vault, including initialization state, lock status, entry count, and Google Drive sync information.

## Usage

```bash
BLANK status
```

## Options

This command has no options.

## Status fields

### Vault information

<ParamField path="Vault" type="string">
  Initialization state:
  - **Initialized**: Vault exists and is ready to use
  - **Not initialized**: No vault found, run `BLANK init`
</ParamField>

<ParamField path="Location" type="string">
  Filesystem path to the vault directory. Typically `~/.blankdrive/vault/` or `%USERPROFILE%\.blankdrive\vault\` on Windows.
</ParamField>

<ParamField path="Status" type="string">
  Lock state:
  - **Unlocked**: Vault is decrypted and accessible
  - **Locked**: Vault is encrypted, requires master password
</ParamField>

<ParamField path="Entries" type="number">
  Total number of entries in the vault (passwords, files, notes). Only shown when unlocked.
</ParamField>

<ParamField path="Created" type="datetime">
  Timestamp when the vault was first initialized. Only shown when unlocked.
</ParamField>

<ParamField path="Last sync" type="datetime">
  Timestamp of the most recent successful sync with Google Drive. Shows "Never" if vault has not been synced. Only shown when unlocked.
</ParamField>

### Google Drive information

<ParamField path="Connected" type="boolean">
  Google Drive connection state:
  - **Yes**: Authenticated and connected
  - **No**: Not authenticated, run `BLANK auth`
</ParamField>

<ParamField path="Mode" type="string">
  Cloud storage mode:
  - **Hidden (appDataFolder)**: Files stored in hidden Google Drive app storage
  - **Public (BlankDrive folder)**: Files stored in visible `BlankDrive/` folder
</ParamField>

<ParamField path="Folder" type="string">
  Public folder path (only shown in public mode). Example: `BlankDrive/vault-backup`
</ParamField>

<ParamField path="Pending" type="string">
  Sync status:
  - **All synced**: No pending uploads
  - **X entries need sync**: Number of entries modified locally but not yet uploaded
  
  Only shown when vault is unlocked and Google Drive is connected.
</ParamField>

## Behavior

### Duress mode detection

If duress mode is active (triggered by a duress password), `status` displays fake information:
- Shows vault as unlocked
- Displays decoy entry count
- Shows fake sync timestamps
- Indicates Google Drive is connected

This protects real vault data under coercion.

### Information availability

Certain fields are only available when the vault is unlocked:
- Entry count
- Creation timestamp
- Last sync timestamp
- Pending uploads count

<Note>
  The `status` command never prompts for a password. It shows available information based on the current lock state.
</Note>

## Use cases

### Quick health check

Verify vault is initialized and unlocked:

```bash
BLANK status
```

### Pre-sync verification

Check pending uploads before syncing:

```bash
BLANK status

```

### Troubleshooting connection

Verify Google Drive authentication:

```bash
BLANK status
```

If `Connected: No`, run:

```bash
BLANK auth
```

### Confirm initialization

After running `BLANK init`, verify setup:

```bash
BLANK status
```

```typescript
export async function statusCommand(): Promise<void> {
  console.log(chalk.bold('\n  BlankDrive Vault Status\n'));

  // Duress mode - show fake status
  if (isInDuressMode()) {
    // ... display decoy information
    return;
  }

  const paths = getVaultPaths();

  // Check if vault exists
  const exists = await vaultExists();
  console.log(`  ${chalk.gray('Vault:')}      ${exists ? chalk.green('Initialized') : chalk.red('Not initialized')}`);

  if (!exists) {
    console.log(chalk.gray('\n  Run "BLANK init" to create a vault.\n'));
    return;
  }

  // ... display vault and sync information
}
```
