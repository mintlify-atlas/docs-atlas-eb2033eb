## Overview

The `audit` command scans all password entries in your vault and provides a comprehensive security report. It checks for expired passwords, weak passwords, and provides strength analysis with actionable recommendations.

<Note>
Audit only analyzes **password entries**. Files and notes are not included in the security audit.
</Note>

## Usage

```bash
BLANK audit [options]
```

## Options

<ParamField path="-a, --all" type="boolean">
  Show all passwords including up-to-date ones. By default, only expired and expiring passwords are shown.
</ParamField>

## Default Behavior

By default, `audit` shows:

1. **Password Age Summary**: Counts of up-to-date, expiring soon, and expired passwords
2. **Password Strength Summary**: Distribution of password strength levels
3. **Weak Passwords**: Passwords with Very Weak or Weak strength (immediate attention needed)
4. **Expired Passwords**: Passwords that exceeded their expiry period
5. **Expiring Soon**: Passwords approaching expiry (75% of expiry period reached)
6. **Recommendations**: Actionable security tips

## Password Age Tracking

### Expiry Period

Each password has an **expiry period** (default: 90 days). This determines when a password is considered:

- **Good**: Age < 75% of expiry period
- **Warning**: Age ≥ 75% of expiry period
- **Expired**: Age ≥ expiry period

### Age Display Format

| Age | Display Format |
|-----|---------------|
| 0 days | Today |
| 1 day | 1 day |
| 2-29 days | X days |
| 30-364 days | X months |
| 365+ days | X years |

### Examples

```
Age: 15 days
Age: 2 months
Age: 1 year
```

### Overdue Calculation

For expired passwords, the report shows how many days overdue:

```
Age: 4 months (32 days overdue)
```

This means the password is 122 days old with a 90-day expiry period (122 - 90 = 32 days overdue).

### Days Until Expiry

For expiring passwords, the report shows remaining days:

```
Age: 2 months (15 days until expiry)
```

This means the password is 75 days old and will expire in 15 days (90 - 75 = 15).

## Prioritization

The audit report prioritizes security issues in this order:

1. **🚨 Weak Passwords** (Very Weak/Weak strength)
2. **⏰ Expired Passwords** (age exceeded expiry period)
3. **⚠️ Expiring Soon** (75%+ of expiry period used)
4. **✓ Up-to-date** (shown only with `--all` flag)

<Tip>
Focus on strengthening weak passwords first, even if they're not expired yet.
</Tip>

## Use Cases


Before security compliance reviews, ensure all passwords meet standards:

```bash

```

### Post-Breach Response

After a security breach, audit to find and update affected passwords:

```bash
BLANK audit
# Update all expired and weak passwords
```

After importing passwords from another source, audit to find weak entries:

```bash
BLANK audit --all
```

## Actionable Workflow

1. **Run the audit**:
   ```bash
   BLANK audit
   ```

2. **Update weak passwords first**:
   ```bash
   BLANK edit "Old Forum Account"
   ```
   Use the built-in generator to create a strong password.

3. **Update expired passwords**:
   ```bash
   BLANK edit "Bank of America"
   ```

4. **Update expiring passwords**:
   ```bash
   BLANK edit "Work Email"
   BLANK edit "GitHub"
   ```

5. **Verify improvements**:
   ```bash
   BLANK audit
   ```

## Recommendations

The audit report includes tailored recommendations:

- **Edit command**: Update passwords with `BLANK edit <entry>`
- **Generate command**: Create strong passwords with `BLANK generate`
- **Weak password priority**: Strengthen weak passwords before expiry
- **Unique passwords**: Use different passwords for each account

## Technical Details

- **Age calculation**: Based on `passwordLastChanged` timestamp (falls back to `created`)
- **Default expiry**: 90 days (customizable per entry)
- **Warning threshold**: 75% of expiry period
- **Strength algorithm**: Based on zxcvbn password strength estimation
- **Entropy calculation**: `log2(charset_size ^ length)`
- **Entry filtering**: Only analyzes entries with `entryType === 'password'`

## Password Last Changed Tracking

Every password entry tracks:

- **`created`**: When the entry was first added
- **`passwordLastChanged`**: When the password was last updated

When you edit a password, `passwordLastChanged` is automatically updated:

```bash
BLANK edit gmail
# Update password
```

The audit uses `passwordLastChanged` (or `created` if not set) to calculate age.
